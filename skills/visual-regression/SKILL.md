# Visual Regression

## 核心理念

**视觉回归测试是 UI 质量的最后一道防线。**

代码测试能验证逻辑正确性，但验证不了用户眼睛看到的东西。
- 布局是否错位？
- 颜色是否正确？
- 元素是否重叠？
- 响应式是否正常？

这些只能通过截图对比来验证。

## 工作原理

```
1. 启动 dev server → 获取 localhost 端口
2. CDP 打开页面 → 截图（基准/当前）
3. 像素级对比 → 标注差异区域
4. 生成视觉差异报告 → 确认是否回归
```

## 前置检查

1. 确保 web-access CDP 可用：
   ```bash
   node "${WEB_ACCESS_DIR}/scripts/check-deps.mjs"
   ```
2. 确保 dev server 运行中，获取端口
3. 确保基准截图存在（首次运行会创建基准）

## 能力模块

### 模块 1: 基准截图创建

**用途**：首次运行或 UI 变更后，建立视觉基准

**流程**：
1. 通过 CDP 打开目标页面（支持多分辨率）
2. 截图并保存到 `output/{version}/test/visual-baselines/`
3. 记录页面 URL、分辨率、截图时间

**支持的分辨率**：
| 设备 | 宽度 | 说明 |
|------|------|------|
| Desktop | 1920px | 桌面标准 |
| Tablet | 768px | iPad 尺寸 |
| Mobile | 375px | iPhone SE 尺寸 |

### 模块 2: 视觉差异检测

**用途**：对比当前截图与基准，检测视觉差异

**流程**：
1. CDP 打开同一页面（相同分辨率）
2. 截图当前状态
3. 像素级对比基准 vs 当前
4. 标注差异区域（diff 图高亮变化部分）

**差异分类**：
| 类型 | 说明 | 严重程度 |
|------|------|---------|
| 布局变化 | 元素位置/大小改变 | critical |
| 颜色变化 | 色值改变 | warning |
| 内容变化 | 文本/图片内容改变 | info |
| 渲染差异 | 抗锯齿/字体渲染微小差异 | ignore |

### 模块 3: UI 组件渲染验证

**用途**：验证特定 UI 组件在不同状态下的渲染

**检查项**：
- [ ] 按钮：默认态、悬停态、点击态、禁用态
- [ ] 表单：空状态、填写态、错误态、提交态
- [ ] 列表：空列表、单条、多条、分页
- [ ] 弹窗：打开、关闭、遮罩层
- [ ] 响应式：各断点下的布局

## 与 web-access CDP 的集成

| 操作 | CDP 命令 | 用途 |
|------|---------|------|
| 截图 | `/screenshot` | 捕获页面渲染状态 |
| 设置视口 | `/eval` → `window.innerWidth` | 模拟不同设备 |
| 滚动 | `/scroll?y=3000` | 触发懒加载内容 |
| 点击 | `/click` | 验证交互态（悬停/点击） |
| 等待 | `/eval` → `waitForSelector` | 等待异步内容加载 |
